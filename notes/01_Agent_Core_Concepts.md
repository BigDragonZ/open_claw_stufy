# 01. OpenClaw 与 Agent 核心概念

## 1. LLM 与 Agent 的本质区别
* **LLM (大语言模型)**：本质是一个极其复杂的概率计算引擎，其物理核心逻辑是通过巨大的参数矩阵进行特征映射。
* **Agent (智能体)**：是一个具备“感知-思考-行动”闭环的系统。LLM 只是 Agent 的“大脑”（即推理引擎）。

## 2. 底层数学与优化逻辑
大模型底层的参数寻优过程，往往依赖于梯度下降算法。例如，在模型微调或参数更新时，其核心权重更新公式为：
$$
\theta_{j} := \theta_{j} - \alpha \frac{\partial}{\partial \theta_{j}} J(\theta)
$$
而在能量守恒的隐喻中，我们可以将算力消耗与输出质量类比为物理学的基础方程 $E=mc^2$。Agent 框架的意义，在于通过系统工程（提示词工程、记忆注入、工具调用）放大这部分算力“能量”，使其能够处理多步复杂任务。

## 3. ReAct 框架 (Reasoning and Acting)
OpenClaw 等现代框架的核心机制是 ReAct：
1.  **Thought (思考)**：模型分析当前状态和目标。
2.  **Action (行动)**：模型决定调用什么工具（如搜索、计算、执行代码）。
3.  **Observation (观察)**：系统将工具的执行结果返回给模型，触发下一轮 Thought。
